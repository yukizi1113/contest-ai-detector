# 🔍 競技プログラミング提出物 AI生成コード判定ツール
## Contest Code AI Detector

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/yukizi1113/contest-ai-detector/blob/main/contest_ai_detector.ipynb)

競技プログラミングコンテストへの提出 Python コードが **AI 生成** か **人間が記述** したものかを判定する Google Colaboratory アプリケーションです。

## 使い方

1. 上の **Open in Colab** バッジをクリックしてノートブックを開く
2. セルを上から順に実行する
3. **③ のセル**で判定したい `.txt` ファイルをアップロードする
4. 判定結果（AI / 人間）と根拠が表示される

複数ファイルの一括判定は、ZIP にまとめて **④ のセル**でアップロードしてください。

## 設計方針

### ❌ 偽陽性（人間 → AI 誤判定）は絶対に許容しない

競技プログラミングにおける提出物には以下のような特徴が見られますが、
これらは **AI 生成の根拠として使用しません**。

| 特徴 | 理由 |
|------|------|
| `n`, `m`, `a`, `b` などの極めて短い変数名 | 競技プログラミングでは標準的 |
| コメントなし / コメントが極めて少ない | 競技プログラミングでは標準的 |
| 説明的な命名規則の欠如 | 競技プログラミングでは標準的 |
| ビジネスコンテキストの欠如 | 競技プログラミングでは標準的 |
| クラス定義（BIT、BinaryTrie 等） | 競技プログラミングでもよく使われる |
| 型アノテーション（`: int`, `-> None` 等） | 競技プログラミングでも使われることがある |

### ✅ 偽陰性（AI → 人間 誤判定）は許容する

判定を厳しくするよりも、**人間が書いたコードを AI と誤判定しないこと**を最優先とします。
AI が生成したコードを「人間が書いた」と判定してしまう場合はあり得ます。

## 検出シグナル

### 強シグナル

| シグナル | スコア | 根拠 |
|---------|--------|------|
| `if __name__ == "__main__":` ガード | +60 | 競技プログラミング提出物には不要 |
| ドキュメント文字列 (docstring) | +50〜+80 | 競技プログラミングでは極めて稀 |
| `from typing import` / `import typing` | +50 | 競技プログラミングでは使用しない |
| `try-except` ブロック | +40 | 競技プログラミングでは不要 |
| `import logging` | +55 | 競技プログラミングでは使用しない |
| `import argparse` | +60 | 競技プログラミングでは使用しない |
| `@dataclass` / `from dataclasses import` | +45 | 競技プログラミングでは使用しない |
| `from abc import ABC` | +40 | 競技プログラミングでは使用しない |

### 中シグナル

| シグナル | スコア | 根拠 |
|---------|--------|------|
| `def main():` + `main()` 呼び出し | +35 | AI 生成コードに多い構造 |
| `__all__` 定義 | +25 | 競技プログラミングでは使用しない |
| `import pathlib` | +30 | 競技プログラミングでは不要 |
| `import json` | +25 | 競技プログラミングではほぼ不使用 |

### 弱シグナル

| シグナル | スコア | 根拠 |
|---------|--------|------|
| 英語の説明的アルゴリズムコメント（3件以上） | +15〜+24 | 人間の競技コードでは稀 |
| `assert` 文（2個以上） | +15 | 競技プログラミングでは通常不使用 |
| 高い空行密度（38%超、20行以上） | +12 | AI は空行を多く挿入する傾向 |

**判定しきい値: 40点**

## 改良点

- 判定ロジックを `contest_ai_detector_core.py` に一元化し、Colab UI と検証コードのドリフトを防止
- `import` / `main()` / `__all__` の検出を AST ベースに寄せ、コメントや文字列に引きずられる偽陽性リスクを低減
- UTF-8 BOM 付き `.txt` を正しくデコードし、Windows 由来のアップロードでも AST 解析が崩れないよう改善
- 検証スクリプトに「人間コード41件で誤判定ゼロ」だけでなく「明らかなAI風コードを実際に検出できるか」のスモークテストを追加

## バリデーション結果

- 41 件の実際の競技プログラミング提出物で **偽陽性 0 件**
- 3 件の明白な AI 風スモークケースを **全件 AI 判定**
- UTF-8 BOM 付き入力のデコードと構文解析を確認済み

## 注意事項

- 本ツールはヒューリスティックベースであり、完全な判定を保証するものではありません
- AI 生成コードでも、上記のシグナルを含まない場合は「人間が記述」と判定されます
- 競技プログラミング以外の用途には適しない場合があります
